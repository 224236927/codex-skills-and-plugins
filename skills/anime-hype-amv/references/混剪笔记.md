# 混剪笔记

这份笔记是给 AI 执行 agent 使用的混剪/燃剪方法入口，不是审美随笔。执行 `anime-hype-amv` skill 或处理类似 AMV/混剪任务时，应先读本笔记，再读项目技法库，再制定剪辑方案。

本笔记负责稳定方法论；项目内实际参考片案例、技法卡、音乐控制记录、小样与 QA 证据应放在 `docs/amv-technique-library/` 与 `artifacts/analysis/`，不要塞回本笔记主体。

## 入口顺序

1. 读本笔记。
2. 查看 `docs/amv-technique-library/index.md`，确认当前项目有哪些案例、技法卡和音乐控制记录。
3. 如果任务涉及参考片，先建立或更新对应的参考片技法库案例。
4. 如果任务涉及具体特效、动效、转场、运镜、剪辑语法、字幕动效或声画关系，按需读取并使用对应技法卡。
5. 如果任务涉及 BGM、节拍、乐句、原声、台词压混或音乐是否成立，读取项目音乐控制记录，并建立本片 phrase map / beat map。
6. 如果某个技法无法一次复现，先做小样与 attempt log，再进入正式成片。

禁止把本笔记当固定流程照抄。每次剪片必须根据当前素材、BGM、目标时长、参考片证据、用户反馈和可验证产物重新选择主风格与辅助手法。

## 建卡硬指标

技法卡不是赏析，也不是灵感摘抄。它必须让新的执行 agent 在没有原作者记忆的情况下复现同类效果。未满足硬指标的卡只能标为候选卡，不能指导正式成片。

### 通用准入

- 有参考证据：参考片时间码、抽帧、密集条或运动条。
- 有自制证明：小样视频、声画样片，或正式成片时间线绑定。
- 有审核结论：人工、Gemini 或模型审美评审的原始结论，必须保留不通过原因。
- 有复现配方：输入素材条件、步骤、参数、工具路线、产物路径、验收标准。
- 有适用边界：适合什么语境，什么素材不能用，常见失败形态是什么。
- 有状态：`candidate / failed / experimental / positive-sample-passed / production-ready`。

只有 `positive-sample-passed` 或 `production-ready` 的卡可以进入正式剪辑决策。`experimental` 可以研究，不能悄悄塞进成片。

### 视效卡硬指标

视效卡必须实际做出小样并送检。合格后要记录核心代码或可运行工程路径，因为 Remotion、HyperFrames、Canvas、shader、FFmpeg/OpenCV 本质上都依赖可复现的代码、命令或参数。

必须包含：

- 实现路线：Remotion / HyperFrames / Canvas-WebGL / FFmpeg-OpenCV / 预渲染遮罩 / 混合路线。
- 核心代码：组件、函数、shader、命令或脚本路径；必要时附关键代码片段。
- 参数范围：持续帧数、强度、速度、模糊、位移、颜色、透明度、粒子密度等。
- 小样路径：正向样片、密集抽帧、失败版 attempt log。
- 审核结论：是否通过、为什么不通过、下一版要改什么。
- 复现步骤：新 agent 从素材到输出应如何一步步做。

### 剪辑卡硬指标

剪辑卡不一定需要代码，但必须有帧级或秒级时间线规则，不能只写“衔接流畅”。

必须包含：

- 镜头结构：A 镜头、桥接、B 镜头，或多镜头队列。
- 切点规则：保留几帧/几秒，在哪个动作阶段切，桥接黑场/白闪/遮挡几帧。
- 运动规则：方向、速度、主体位置、景别、姿态完成度。
- 素材检索条件：新 agent 应该去找什么动作、表情、镜头类型。
- 小样或绑定：至少有自制样片，或正式成片中明确时间码绑定。
- 失败条件：方向错、动作阶段错、桥接过长、主体跳位、节奏机械等。

### 音乐卡硬指标

音乐卡不能只写“燃”。它必须复现声画关系。

必须包含：

- phrase map：intro、build、pre-drop、drop、break、ending 等段落。
- beat / transient map：强拍、鼓点、静音、反拍、瞬态点。
- peak owner：最强音乐点归属哪个画面事件。
- 原声策略：保留哪句台词，哪里 duck，哪里静音，哪里加 SFX。
- 响度目标：整体响度、峰值、台词段 BGM 压低幅度。
- 生成信息：若 BGM 是生成的，记录模型、提示词、任务 id/seed、取用版本和废弃版本原因。
- 正向样片：10-20 秒声画样片，或正式成片时间码绑定。

## 证据语言

所有判断必须落到可观察、可验证、可复查的证据。不要直接把“舒服、好看、成熟、有感觉、廉价、爽、燃、氛围好”等词作为执行依据。

允许引用用户原话，但必须转译为证据条件：

| 主观说法 | 执行转译 |
|---|---|
| 有感觉 | 有可见段落语法、动效层级、声画落点、情绪递进和结尾回收 |
| 很燃 | 强视觉事件落在强音乐节点上，并有蓄力、释放、结果或反应 |
| 高级 | 效果有适用语境、层级、遮罩/纹理/运动依据，不是全片平铺滤镜 |
| 像参考片 | 有事件台账、技法卡绑定、音乐结构对照和抽帧证据 |
| 音乐不行 | peak 不落视觉爆点、段落不清、抢台词、能量曲线平、迫使画面乱切 |

可用证据包括：用户明确反馈、参考片抽帧、候选素材接触表、波形/频谱/动态范围、BGM 段落结构、动作方向、角色表情变化、台词语义、字幕对齐、完整解码、音量峰值、重复镜头比例、技法小样和最终成片抽帧。

## 参考片技法库

参考片不是拿来“嘴上学习”的。只要用户要求参考、借鉴、学习、做出原片感觉、建立库或对比参考片，就必须创建或更新项目级参考片技法库案例。

### 案例必须包含

- 基本信息：标题、来源、目标、参考视频路径。
- 证据位置：抽帧、密集条、波形、对照图、小样目录。
- 事件台账：时间段、画面变化、叙事功能、技法候选、音乐/原声触发、证据、置信度。
- 技法候选池：视觉特效、动效字幕、转场、运镜、剪辑语法、声画关系、素材选择规则。
- 音乐结构：reference phrase map、主要 hit 点、source audio 或字幕语义节点。
- 返工绑定规则：目标成片的哪些时间线节点会借用哪些卡。
- 不通过条件：只列指标、无证据帧、无技法卡、无小样重试、无成片绑定。

### 抽帧层级

| 层级 | 推荐方式 | 目的 |
|---|---|---|
| 全片结构 | 1 fps contact sheet | 看段落、情绪曲线、素材语义 |
| 高能段 | 6-12 fps strip | 看转场、闪白、遮罩、字幕、运镜 |
| 复杂技法点 | 逐帧或接近逐帧 | 拆效果层级、遮罩、切点、纹理 |
| 音乐 | waveform + beat/phrase map | 判断声画落点和能量曲线 |

### 技法卡类型

- 视觉特效卡：液态遮罩、RGB split、glitch、displace、film grain、纹理、glow、光泄漏、冲击帧。
- 动效/字幕卡：标题、logo、字词入场、字幕压迫、符号闪现、文字作为节拍。
- 转场卡：白闪、黑场、遮挡、匹配动作、动模糊切、mask reveal、smash cut。
- 运镜/运动卡：push-in、pull-back、whip pan、手持震动、snap zoom、speed ramp。
- 剪辑语法卡：同向动作衔接、姿态递进、视线链、反应链、平行蒙太奇、母题回环。
- 声画卡：鼓点白闪、原声刺点、静音断裂、歌词/字幕锁定、duck-and-return。
- 素材选择卡：什么素材能支撑技法，什么素材不能用。

### 小样与重试

复杂技法不能只描述。复现路线按 v0-v3 走：

- v0：机制能不能成立。
- v1：参数能不能调顺。
- v2：审美上是否接近参考或成为可用替代。
- v3：换成目标素材后是否仍成立。

三次认真尝试仍失败，就记录失败原因，收窄适用范围或换视觉等效方案，不要把半成品塞进正式片。

## 主风格一致性

一支短片只能有一个主风格，最多 1-2 个辅助段落功能。主风格由素材和 BGM 共同决定。

| 输入证据 | 主风格候选 | 不应选择 |
|---|---|---|
| 强 drop、释放/撞击素材多 | 动作冲击型、旋律动作型 | 长台词推进型 |
| 有可靠台词、脸部反应、BGM 可 duck | 台词/字幕推进型 | 纯高速硬切 |
| 多角色多地点同时行动 | 平行蒙太奇型、群像回收型 | 单角色独白式结构 |
| 反派/威胁/暗色符号占主导 | 暗色威胁型、母题回环 | 明亮庆祝型 |
| 动作少但关系/回忆强 | 关系/记忆低密度型、抒情桥接型 | 主打撞击的动作冲击型 |
| 结尾需要结果证明 | 群像/城市/结果回收型 | 新开无关冲突风格 |
| 用户要求玩梗或节庆 | 奖励式结尾型 | 严肃威胁型 |

风格一致性靠锚点维持：

- 音乐锚点：同一 BGM 段落或可解释的编曲变化。
- 视觉锚点：重复但受控的母题、颜色、光源、动作方向、空间类别。
- 语义锚点：同一命题持续推进，例如压力 -> 回应 -> 行动 -> 结果。
- 技法锚点：重复使用同一类卡，但每次承担不同功能。

## 烟季十章风格库

烟季参考片不是模板，而是“音乐章节对应段落语法”的风格库。不得复制其题材、BGM、章节长度、节日结尾或字幕密度，只迁移选择条件和画面组织方式。

### 1. 大规模开场 / 身份宣言 / 战斗铺场

可用条件：需要建立世界级危机、阵营、主要冲突或强主题命题；素材具备大场面、威胁源、角色亮相、武器/能量释放、宣言或标题感画面。

镜头语法：大范围冲突 -> 角色脸/眼 -> 武器/能量 -> 冲击释放 -> 宣言式画面。宽景、近脸、武器、能量爆发、轮廓画面交替，避免连续同类镜头。

效果边界：允许高对比、短白闪、强光源、颜色对撞；只在冲击、亮相、段落切换处使用。

### 2. 旋律型快节奏动作蒙太奇

可用条件：BGM 有稳定旋律推进和清楚节拍；素材有动作启动、追逐、转身、表情变化、碰撞、场景颜色差异。

镜头语法：手/脚/身体启动 -> 脸部确认 -> 追逐/碰撞 -> 颜色或空间切换。切点优先落在动作方向变化、身体启动、亮度突变或鼓点上。

效果边界：允许短白闪、速度模糊、轻 punch zoom；不要每刀固定使用。

### 3. 暗色威胁 / 反派态度段

可用条件：素材以敌方、威胁、黑红紫光源、眼睛、面具、阴影脸、暴力结果为主；BGM 更低频、更重鼓点、更少明亮高频。

镜头语法：威胁符号 -> 反派脸/眼 -> 受害结果 -> 暴力动作 -> 短暂反差插入。少用英雄宽景，多用近脸、遮挡、阴影、局部动作。

效果边界：允许 RGB/glitch、红黑色调、阴影强化、短 white-out；不要全片同一暗色滤镜。

### 4. 台词/字幕驱动推进段

可用条件：有可靠原声台词、字幕或宣言句；画面能提供脸、困境、努力、支援、抬头、突破等证据；BGM 可承托人声。

镜头语法：困境/疲劳 -> 挑战/质问 -> 支援/回应 -> 行动准备 -> 释放/突破。台词段镜头停留要长于动作冲击段。

效果边界：少用重特效；允许光斑、短白闪、运动模糊；不在长台词上叠高速战斗切。

### 5. Pop / 喜剧 / 呼吸段

可用条件：高压段后需要恢复信息；素材有表情反应、轻动作、群体关系、短 gag、低风险画面。

镜头语法：表情 -> 简单动作 loop -> 反应 -> 关系信息。节奏仍对齐音乐，但镜头功能从冲击转为信息恢复。

效果边界：减少重闪、震动、暗色滤镜；动作刺激型短片只适合微呼吸。

### 6. 肢体动作 / Gesture / Gag 节奏段

可用条件：素材有可匹配的手势、指向、转身、跳跃、挥刀、身体启动、重复 pose；BGM 有重复节拍或舞曲型律动。

镜头语法：按动作姿态匹配，不按剧情顺序。手伸出接手伸出，身体旋转接方向旋转，拔刀接出手，能量形成接释放。

效果边界：保持动作轮廓清楚，少用遮挡动作的滤镜。

### 7. 关系 / 记忆 / 低密度段

可用条件：素材有脸、手、天空、背光、回忆、失去、重逢、承诺等低密度画面；BGM 有旋律短语或歌词落点。

镜头语法：脸/手/光源 -> 关系证据 -> 低密度动作 -> 语义回收。镜头停留足够让表情或关系成立。

效果边界：允许柔光、白 fade、light leak、轻 bloom；不塞进最硬 beat cluster，除非作为极短语义钉子。

### 8. 抒情 / 光色桥接段

可用条件：素材有日常、阳光、雨、房间、烟火、奔跑、手、微表情；需要通过光源、颜色、方向连接跨来源镜头。

镜头语法：以光色连续、动作方向、构图相似连接镜头，而不是依赖动作冲击。每刀都应提供光源变化、人物关系或空间信息。

效果边界：允许暖色 bloom、软转场、柔和曝光变化；不是无功能慢镜头填充。

### 9. 群像 / 时代记忆 / 庆祝回收段

可用条件：素材有群体、城市、烟火、共享视线、抬头、归来、庆祝或结果画面；需要从冲突进入结果回收。

镜头语法：个体结果 -> 群体回应 -> 城市/空间结果 -> 共享视线或回归。

效果边界：允许烟火 bloom、色彩增强、柔和光效；必须服务“结果已发生”的语义。

### 10. 节庆 / 玩梗 / 奖励式结尾

可用条件：片子允许强风格转向、玩梗或节庆奖励；目标平台和题材支持明显的结尾变化。

镜头语法：持续装饰框架、红金画面、喜剧脸、直接 gag timing。它成立的前提是全片主要结构已经完成。

效果边界：普通燃剪不要直接复制；只有结尾语义明确允许风格突变时才使用。

## 通用镜头语言工具箱

以下是可迁移模块，不是固定顺序。

- 母题回环：同一视觉符号少量重复，每次上下文变化。第一次建立压力，第二次加重或被打破，结尾反向回收。
- 平行蒙太奇：不同地点、角色或战线按同一语义方向并列。连续镜头之间要有动作、颜色、方向、光源或语义递进关系。
- 动作预备与释放：预备镜头稍长，释放镜头短、准、重。音乐重音压在释放或命中上，不压在准备动作上。
- 视线与反应链：用“看见 -> 承受 -> 反应 -> 行动”建立因果。反应镜头必须推动下一刀。
- 颜色与光源桥接：用颜色、亮度、光源方向或质感连接跨来源镜头，不用统一滤镜掩盖素材重复。
- 留白后爆发：主高潮前短暂降低密度，提供人物、关系或空间信息，然后用下一段爆发形成落差。
- Callback 收束：结尾回应开头或中段的视觉/语义问题，可以回到相反含义。
- 同向动作转场：前后镜头动作方向一致，中间可用 1-3 帧黑场、白闪、遮挡或冲击帧。
- 姿态递进剪辑：多个角色做同类动作时，按动作完成度排序，制造一个集体动作。

## BGM 选择与处理

音乐是全片时间骨架，应单独记录，不混进单张技法卡。

1. BGM 选择要服从证据，不能只追求“更炸”。
   - 如果用户明确否定某首 BGM，必须换思路或重做。
   - 换 BGM 必须带来明确收益：更匹配语义曲线、更清楚段落、更好踩重音、更稳尾奏、更适合人声混入。
   - 对燃剪来说，段落稳定、重音清楚、不抢人声、高频不过分集中、能支撑连续画面变化的音乐，常常比更吵的音乐更可用。
2. 每支片建立 phrase map。
   - 标出 intro、first lift、tension bed、pre-drop、drop、peak、break、ending。
   - 不把音量高低等同于结构；编曲换层、停顿、旋律转向也是结构点。
3. 每支片建立 beat / hit map。
   - 标出 kick、snare、clap、bass transient、silence gap、fill、vocal syllable、source line ending、impact SFX。
   - 最强视觉事件必须拥有最强音乐节点。
4. 台词下的 BGM 必须 duck。
   - 台词段保持底层能量，但不得盖住原声内容。
   - 台词尾音或停顿后，BGM 在下一个视觉节点恢复。
5. BGM 不通过条件。
   - peak 无法落在最强视觉事件。
   - 段落变化不清。
   - 动态太平。
   - 抢台词或字幕。
   - 迫使剪辑重复、乱切或提前高潮。

## 台词与字幕规则

- 原声内容、画面含义、字幕文字必须一致。
- 台词只有在能完成明确语义功能时才出现：宣告、否定、质问、鼓动、转折、收束。
- 台词段要给可读画面：脸、姿态、困境、抬头、反应、静止压力。
- 高速动作镜头不承载长台词。
- 每个台词段至少检查一帧字幕/口型/画面含义是否对齐。
- 如果字幕不同步，先修 clip-local retiming，不用特效遮问题。

## 特效与画面处理规则

特效是局部标点，不是整片皮肤。画面强度优先来自镜头状态：表情改变、身体启动、武器出手、能量释放、环境破碎、光源爆发、角色位置反转。

可用特效：

- 2-4 帧 white flash。
- 2-3 帧 RGB split。
- release frame punch zoom。
- windup slower -> release faster speed ramp。
- 方向性 blur。
- 有光源依据的 bloom。
- silence break black/white frame。
- mask reveal / liquid dissolve / displacement 类效果，但必须有小样或卡片依据。
- 字幕/标题动效，但必须承担语义或节拍功能。

避免：

- 连续震动。
- 全片同滤镜。
- 每刀同样闪光。
- 普通脸部自发光。
- 遮挡字幕、眼神、手、武器、撞击点。
- 无光源依据的 glow。
- 没有音乐或语义触发的高级特效。

光、雨、雪、烟、火花等粒子必须匹配画面方向、速度、密度、深度和持续时间。

## 制作流程护栏

1. 剪前必须看素材。
   - 建候选接触表。
   - 抽字幕或关键词。
   - 标记候选场景：恐惧、压力、决定、揭示、攻击、撞击、反转、群体反应、余波。
   - 排除片头片尾、标题卡、空白转场、静态说明、重复片段、只有画面显著但无段落功能的镜头。
2. 先选语义命题，再选镜头。
   - 命题决定镜头功能：setup、proof、escalation、impact、release、payoff。
   - 每个 clip 必须能说明自己的段落功能。
   - 除非有母题回环等明确理由，不重复使用相近源时间。
3. 先 clean cut，再加效果。
   - 第一轮验证音乐节点、镜头语义、动作状态。
   - 第二轮只在已标记节点添加局部效果。
   - 第三轮抽帧和局部密集检查。
4. 成片必须绑定技法库。
   - 关键节点要写明使用哪张技法卡或哪个案例规则。
   - 没有卡的关键技法要补卡或写一次性例外。
5. 输出必须可验证。
   - 文件名唯一。
   - 记录路径、时长、大小、SHA256、视频/音频格式、contact sheet 路径。
   - 完整解码通过。
   - 全片 loudness 和高潮段 loudness 至少各检查一次。
   - 结尾 fade 或 audio resolve 必须发生在时间线内。

## 案例引用原则

本笔记不积累每个用户认可样片的细节。认可样片只能作为外部校准材料，除非它产生新的泛化规则。

允许加入主文档：

- 新风格类型。
- 新选择条件。
- 新验收门槛。
- 新失败模式。
- 新的 BGM/台词/特效处理规则。
- 新的技法库组织规则。

不应加入主文档：

- 每个样片的完整结构。
- 每个样片的文件路径、SHA、片段数量。
- 只对单一素材成立的时间轴。
- 用户单次称赞但无法泛化的表达。

样片细节应进入 `docs/amv-technique-library/cases/`；可复用技法进入 `docs/amv-technique-library/techniques/`；重证据进入 `artifacts/analysis/`。

## 交付前最低自检

- 这支片的语义命题是什么？
- 主风格是哪一种？辅助风格是哪 1-2 种？分别服务什么结构功能？
- 参考片案例是否已建立或更新？
- 使用了哪些技法卡？每张卡绑定到哪些时间线节点？
- 有没有复杂技法的小样或 attempt log？
- 音乐有哪些结构变化，而不是只有哪里更响？
- 最强音乐节点对应哪一个最强视觉事件？
- 是否跨素材库按意义选镜头，而不是按集数顺序偷懒？
- 是否有一段提供人物、关系或空间信息的低密度镜头？
- 是否有结尾回收开头或中段命题？
- 台词是否少、准、对齐？
- 特效是否只出现在该出现的地方？
- 抽帧板是否能看出新版不是旧版换皮？
- 局部高能段是否避免连续重复同一动作状态？
- 成片是否完整解码、音量不过载、文件信息可复查？
