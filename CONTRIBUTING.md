# Contributing

Thanks for improving this repository.

## Principles

- Keep beginner-facing docs readable.
- Prefer practical starter assets over empty placeholders.
- Keep skills concise and searchable.
- Keep plugin manifests valid and obvious to adapt.

## Repo Conventions

- Skills live under `skills/`
- Plugins live under `plugins/`
- Repo marketplace metadata lives under `.agents/plugins/`
- Shared visual assets live under `assets/`

## For Skill Changes

- Keep `SKILL.md` focused on when to use and how to execute.
- Validate skill structure when possible before publishing changes.
- Add references only when they reduce repetition or improve reliability.

## For Plugin Changes

- Keep `.codex-plugin/plugin.json` present.
- Use relative paths beginning with `./`.
- Keep starter manifests easy to fork, not over-customized.

## Pull Request Style

- Explain what changed.
- Mention whether the change affects skills, plugins, docs, or layout.
- Include screenshots when the README visual presentation changes noticeably.
